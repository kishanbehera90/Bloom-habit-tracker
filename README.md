# Thrive — Habit & Wellness Dashboard

Full-stack personal habit tracker: Express + Postgres backend, vanilla HTML/CSS/JS frontend, username/password login, daily to-do list, dark mode.

## Run locally

```bash
cd habit-tracker
npm install
cp .env.example .env   # fill in SESSION_SECRET and DATABASE_URL
npm start
```

`DATABASE_URL` must point to a Postgres database (e.g. a free [Neon](https://neon.tech) project). Open http://localhost:3000 — you'll be redirected to the sign-in page. Create an account, then start adding habits.

## Deploy to Render (free tier)

1. Push this `habit-tracker` folder to a GitHub repo.
2. In Render, click **New > Blueprint** and point it at the repo (it reads `render.yaml` automatically), or create a **Web Service** manually with:
   - Build command: `npm install`
   - Start command: `npm start`
3. Set environment variable `DATABASE_URL` to your Postgres connection string (Render's free tier doesn't support persistent disks, so an external Postgres like Neon is required to keep data across restarts/redeploys).
4. `SESSION_SECRET` is auto-generated by the blueprint.
5. Deploy. Render gives you a public HTTPS URL you can open from your phone or any browser — bookmark it for daily use.

## Deploy to Railway / Fly.io

Same idea: `npm install && npm start`, with `DATABASE_URL` and `SESSION_SECRET` set as environment variables. No persistent volume needed since data lives in Postgres.

## Notes

- This app supports multiple user accounts — each user only sees their own habits, to-dos, and history.
- Sessions are stored in Postgres (`connect-pg-simple`) and last 30 days.
- Free-tier Postgres providers (like Neon) don't expire data, but free Render web services spin down after ~15 min idle — first request after that takes up to ~50 seconds to wake up.
